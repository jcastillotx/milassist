# Workflow

## 1. Discovery

- Read the issue and current code.
- Identify constraints and dependencies.
- Enter plan mode and ask all required intake questions.
- Do not modify files until intake questions are answered.

## 2. Planning

- Propose a minimal, testable change.
- Define acceptance criteria.
- Confirm deployment target and stack choices are captured.

## 3. Implementation

- Apply small, focused commits.
- Keep formatting and style consistent.

## 4. Verification

- Run relevant tests.
- Update docs if behavior changes.

## 5. Release Readiness

- Ensure changelog and versioning are updated when required.
